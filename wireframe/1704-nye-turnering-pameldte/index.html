Anbefaling: Option 1 — Field List som primær, med Options 2 og 4 som view-toggle

Field List er den eneste visningen som dekker ALLE turneringstyper (slag, MP, stableford) og er det formatet 100% av spillere allerede kjenner fra GolfBox. Den løser grunnbehovet ("hvem er påmeldt") og gir klar struktur for AK-elev-markering som er en nøkkelforretningsmekanisme (trenere ser raskt egne elever, spillere ser "folk fra min gjeng").

Implementasjonsflyt:

  • Fase 1: Lever Option 1 Field List som default på /portal/turneringer/[id]/pameldte.
  • Fase 1: Legg view-tabs "Liste / Tee-tider / Leaderboard" med Option 2 og 4 i samme side — samme URL, querystring-styrt (?view=tee).
  • Fase 2: Bracket (Option 3) kun synlig for Match Play-turneringer (filter på Tournament.format = MATCH_PLAY).
  • Fase 2: Player Cards (Option 5) som hover-utvidelse på rad — reduserer kompleksitet, behold data-krav.

Teknisk: GolfBox API → Registration (field), StartList (flights/tee-tider), Scoring (live leaderboard). Cache Registration 5 min, StartList 1 min ved aktiv turnering, Scoring 30s når "Live"-state er aktiv. AK-elev-markering gjøres via join Registration.email → User.email → StudentRelation.
