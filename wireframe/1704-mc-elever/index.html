UX-notater

  • Grid med 3 kolonner gir visuell gjenkjenning — avatar + navn først, kategori-badge som fargekode i hjørnet.
  • Filter-row fikserer A-K kategorier som pills — Anders kan hurtig filtrere sine toppspillere (A/B) vs bredde (C-K).
  • KPI-strip under filter gir helikopter-tall (42 aktive, 148 økter, 7 nye) — uten å forlate siden.
  • Kort med 32+ dager inaktivitet får gul varselfarge i foot — tydelig follow-up-signal.
  • "+ Ny elev" CTA pinned til høyre i filter-row — alltid synlig selv ved scroll.

Anbefaling: Option 1 — Grid Cards

Grid Cards balanserer visuell gjenkjenning (avatar + navn + kategori-badge) med effektive filtere og KPI-strip. For Anders er daglig "finn en elev"-bruksmønster hyppigst, og kort er mer skannbart enn tabellrader når vi har 42 aktive elever. Data Table (Option 2) er en sterk toer og bør være tilgjengelig som toggle når bulk-oppgaver eller eksport dominerer.

På lengre sikt (100+ elever) bør Segment Dashboard (Option 3) integreres som separat "Analyse"-view, og Pipeline (Option 5) som egen win-back-funksjon i stedet for hovedside.

Når velge hvilket

  • Option 1 (Grid Cards): Default. Best for visuell bruk og <50 aktive.
  • Option 2 (Data Table): Når bulk-aksjoner (SMS, eksport, planendring) dominerer arbeidsflyten.
  • Option 3 (Segment): Strategisk månedlig/kvartalsvis porteføljegjennomgang.
  • Option 4 (Map): Reise-planlegging eller regional ekspansjon.
  • Option 5 (Pipeline): Når retention og win-back blir fast operativ rutine.
