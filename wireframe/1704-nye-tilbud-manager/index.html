UX-notater

  • Mål-først flyt: Anders velger "hva" (Winback, Nye kunder, Sesong) før "hvordan" — reduserer beslutnings-tretthet.
  • AK-anbefaling-boks gir datadrevne forslag (100 kr er typisk winback-nivå) og historisk konverteringsrate.

  • Forhåndsvisning-boks lest som "Kode X gir Y på Z" — eliminerer tvetydighet før publisering.
  • Best for admin som lager 2–5 nye kampanjer i måneden. Svakere for daglig overvåking av aktive koder.

Anbefaling: Option 1 — Code Table (SAFE)

Option 1 er default fordi Anders primært trenger daglig oversikt over hvilke promo-koder som er i bruk og hvor mye igjen det er på hver. Tabellen skalerer til 50+ koder, er enkel for nye admins, og matcher samme mønster som Epostkampanjer (N10) og Bookinger (N2). Rad-handlinger (Rediger, Pause, Dupliser) dekker 95 % av hverdags-arbeidet.

Option 5 (Dynamic Pricing) bør bygges som separat undermeny under "Prising" fordi det løser en helt annen oppgave — løpende prislogikk, ikke enkeltkoder. Ikke forsøk å kombinere koder og regler i samme tabell.

Option 3 (Builder Focus) bør bli ny-kampanje-modal som åpnes fra Option 1 sin "+ Ny promo-kode"-knapp. Mål-først-wizard er best som dedikert flyt, ikke daglig dashboard.

Option 4 (Performance) kan bli egen fane (Tilbud → Ytelse) for månedlig rapportering.

Når velge hvilket

  • Option 1 (Code Table): Default daglig arbeidsflate. Best for å skanne alle aktive koder.
  • Option 2 (Campaign Cards): Hvis vi vil fremheve 5–10 samtidige kampanjer visuelt, f.eks. på mobil eller sekundært dashboard.
  • Option 3 (Builder Focus): Som modal fra "+ Ny promo-kode". Fokus på mål, ikke felter.
  • Option 4 (Performance): Egen fane for månedsrapport — ROI, funnel, topp 5.
  • Option 5 (Dynamic Pricing): Egen undermeny under Prising — regelbasert prissetting, ikke engangskoder.
