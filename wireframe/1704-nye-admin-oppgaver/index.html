UX-notater

  • Klassisk Trello/Linear-kanban — tre kolonner (To Do / In Progress / Done) er det mest gjenkjennelige og lavt læringspåslag.
  • Hvert kort viser prio-tag, relatert entity (elev eller gruppe), domene-tag (coaching, fakturering, lead, innhold), assignee og due date.
  • Forsinket-kort har sterk kontrast på due-date for tydelig varsel.
  • KPI-strip øverst gir dagens status på ett blikk før man dykker ned i kort.
  • Filter-pills filtrerer alle tre kolonner samtidig (Mine / Coaching / Fakturering / Leads / Innhold).

UX-notater

  • Seksjonsgrupperinger (Forsinket / I dag / Denne uka / Senere) gjør listen scrollbar på en logisk måte.

UX-notater

  • Månedskalender gir visuell oversikt over hvor i måneden det er travelt — nyttig for langsiktig planlegging.

UX-notater

  • Hero-banner med «4 oppgaver i dag» er dagens første sanity-sjekk — matcher ADHD-vennlig «én ting om gangen»-filosofi.

  • «Alle andre»-seksjon krymper senere-oppgaver til bakgrunnen — reduserer visuell støy.
  • Sterk for Anders som daglig driver — svakere for teammedlemmer som må se hele bildet.

UX-notater

  • Gruppering etter domene (Coaching / Fakturering / Lead / Innhold / Drift) speiler hvordan Anders allerede tenker — bytter «modus» per blokk.
  • Toggle lar deg bytte gruppering (Domene / Elev / Prosjekt / Prioritet) uten å miste data — bare reorganiserer.
  • Hver gruppe har egen «forsinket»-badge og teller, så man vet hvilken domene som brenner mest.
  • Passer best når Anders vil jobbe fokusert i ett domene om gangen (f.eks. «i dag fokuserer jeg på fakturering»).
  • Svakere for rent daglig drift — krever mer scrolling sammenlignet med Today Focus eller Kanban.

Anbefaling: Option 1 — Kanban Board

Kanban er det mest gjenkjennelige mønsteret (Trello, Linear, Jira) og gir tydelig visuell status-oversikt med lav læringskurve. De tre kolonnene (To Do / In Progress / Done) passer AdminTask-modellen direkte, og kort med prio, due, relaterte entities, domene-tag og assignee dekker alle kjernefeltene. Drag-to-status-workflow er intuitiv og støtter både enkeltbruker (Anders) og fremtidig multi-bruker (hvis flere coacher får admin-tilgang). Option 4 (Today Focus) er en sterk toer for daglig arbeidsflyt — vurder å legge den til som en sekundær visning (toggle) senere. Option 5 (Client-Grouped) er nyttig som alternativ visning når Anders vil jobbe fokusert i ett domene om gangen.

Når velge hvilket

  • Option 1 (Kanban): Default. Best for blanding av enkeltbruker og fremtidig team.
  • Option 2 (Prio List): Hvis informasjonstetthet og keyboard-hastighet er viktigst.
  • Option 3 (Calendar): Hvis hovedbruken er langsiktig planlegging (måned-visning).
  • Option 4 (Today Focus): Hvis Anders er eneste bruker og daglig rutine er kritisk.
  • Option 5 (Client-Grouped): Hvis domene-bryter-flyt dominerer (f.eks. «fakturering-dag»).
