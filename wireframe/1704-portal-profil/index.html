UX-notater

  • Alt synlig uten klikk — profil-header + KPI + achievements + HCP + aktivitet.
  • Achievements grid har rarity-farger (Common/Rare/Epic/Legendary) for gamification.
  • HCP-historikk som sparkline + tre datapunkter gir rask kontekst.
  • Anbefalt default — skalerer fra ny elev (lite data) til veteran (fylt dashboard).
  • Mobilvennlig: grid kollapser til én kolonne, KPI til 2x2.

UX-notater

  • Editorial-stil — stor portrett, fortelling, signatur-stats.
  • Fungerer godt som delt lenke (sosialt, Instagram-story).

Anbefaling: Option 1 — Dashboard

Dashboard er tryggest — profil-header + KPI + achievements + HCP-historikk + aktivitet gir alt på ett skjermbilde uten læringsbarriere. Passer både nye elever (lite data, men åpen struktur) og veteraner (fylt dashboard). Skalerer på mobil ved å stable grid. Option 3 (Showcase) er sterk toer for delbar merkevareopplevelse. Option 5 (Social) kan bygges inn som et sosialt kort i Dashboard, eller som egen underside.

Når velge hvilket

  • Option 1 (Dashboard): Default. Best for alle brukertyper.
  • Option 2 (Tabs): Hvis spilleren har mye data og trenger dypdykk.
  • Option 3 (Showcase): Hvis profil skal deles eksternt (social proof, merkevarebygging).
  • Option 4 (Linear List): Hvis tetthet og skanning er viktigst (power-user, admin).
  • Option 5 (Social): Hvis sosial/gamification-dimensjon er primærmotivasjon.
