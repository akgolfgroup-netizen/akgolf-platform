## Felles strategi

For hver oppgave folger vi samme monster:
1. **Dypanalyse av mockup** — les HTML 1:1, identifiser alle visuelle elementer
2. **V2-fil pa siden av V1** — ikke rriv eksisterende, bygg parallel
3. **Aktivering via cookie + query** — `?v=2` setter cookie i 30 dager, V1 rives nar verifisert
4. **Brand Guide V2.0 tokens kun** — `--color-primary`, `--color-accent`, `--color-surface`, etc.
5. **Server Components der mulig** — kun "use client" der reell interaktivitet
6. **Maks 300 linjer per fil** — splitt seksjoner i egne komponenter
7. **TS strict + ESLint 0 errors** for hver commit
8. **Real data der mulig** — mock kun for nye datakilder

## Aktiverings-monster (felles)

For hver V2:
1. Sett feature-flag i `proxy.ts`:
   ```ts
   const TRENINGSPLAN_V2_ENABLED = process.env.TRENINGSPLAN_V2_ENABLED === "true";
   const TRENINGSPLAN_V2_COOKIE = "treningsplan_v2";
   ```
2. `?v2=1` query setter cookie i 30 dager
3. `?v2=0` fjerner cookie
4. Global flag sla pa for alle nar V2 er bekreftet
5. Etter 1 uke prod-test uten klager: bytt default + rriv V1

Samme monster som BOOKING_V2_ENABLED i `proxy.ts:27-28`.
