Design Seksjon 6: Datamodell

Nye Prisma-modeller for Coach Hub

Database Schema — Unified Inbox
UnifiedMessage
id String @id
channel Channel // EMAIL, INSTAGRAM, MESSENGER, WHATSAPP, IMESSAGE
direction Direction // INBOUND, OUTBOUND
externalId String // ID fra kildesystem
senderName String
senderHandle String // email/handle/telefon
subject String?
content String
receivedAt DateTime
status MessageStatus // PENDING, AI_READY, SENT, FAILED
assignedTo User?
threadId String?
aiResponse AIResponse?
AIResponse
id String @id
messageId String @unique
draftContent String
finalContent String? // etter redigering
confidence Float // 0.0 - 1.0
category String // pris, booking, info, etc
modelUsed String // claude-sonnet, ollama, etc
autoSent Boolean
wasEdited Boolean
editDiff String? // hva ble endret
approvedBy User?
approvedAt DateTime?
AILearning
id String @id
userId String // hvem AI-en lærer fra
category String
pattern String // input-mønster
response String // godkjent svar
confidence Float
usageCount Int // antall ganger brukt
lastUsed DateTime
corrections Json // historikk av endringer
PushSubscription
id String @id
userId String
endpoint String
p256dh String
auth String
deviceType String // iphone, android, ipad, desktop
createdAt DateTime

Nøkkelpunkter

  • UnifiedMessage: Alle meldinger fra alle kanaler i én tabell
  • AIResponse: AI-utkast + redigeringer + læring
  • AILearning: Mønstre AI-en har lært over tid
  • PushSubscription: Web Push for alle enheter
