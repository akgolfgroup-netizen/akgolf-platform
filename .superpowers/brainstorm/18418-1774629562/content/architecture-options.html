Arkitektur for Coach Hub

Tre tilnærminger — velg basert på hastighet vs. fleksibilitet

A

Alt-i-ett (Anbefalt)

Coach Hub bygges inn i eksisterende akgolf-website

Fordeler

  • Delt kodebase, én deploy
  • Samme auth-system (Supabase)
  • Deler komponenter med Player Hub
  • Raskest å bygge (2-3 uker)

Ulemper

  • Større bundle-størrelse
  • Alle funksjoner i samme repo
B

Separat App

Coach Hub som eget Next.js-prosjekt

Fordeler

  • Helt isolert kodebase
  • Kan deployes uavhengig
  • Lettere bundle

Ulemper

  • Duplisert kode (auth, komponenter)
  • To repos å vedlikeholde
  • Lengre utviklingstid (4-5 uker)
C

Monorepo

Turborepo med delte pakker

Fordeler

  • Beste av begge verdener
  • Delt kode i pakker
  • Separate deploys
  • Skalerbart for fremtiden

Ulemper

  • Kompleks oppsett
  • Må migrere eksisterende kode
  • Lengst utviklingstid (5-6 uker)

Min anbefaling: A (Alt-i-ett)

Du har allerede en portal i akgolf-website. Coach Hub blir en ny route-gruppe ved siden av spillerportalen:

Mappestruktur
app/
├── portal/(dashboard)/ → Player Hub (eksisterende)
├── coach/(dashboard)/ → Coach Hub (ny)
│ ├── inbox/ → Unified inbox
│ ├── chat/ → AI-chat
│ ├── players/ → Spilleroversikt
│ ├── sessions/ → Coaching-modus
│ └── approvals/ → Godkjenningskø
└── api/
    ├── coach/ → Coach Hub API
    └── integrations/ → Gmail, Meta, WhatsApp, iMessage
